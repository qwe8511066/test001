
**快速活体检测人脸识别及智能防撞报警综合系统升级实施方案**

<a name="heading_0"></a>**一、升级概述**

本次升级针对现有“快速活体检测人脸识别综合系统及智能防撞报警综合系统”进行功能优化，核心目标是完善人脸识别的活体检测能力，解决升级前仅能实现基础人脸识别+眨眼活体检测、活体校验维度单一的问题。升级后，系统在保留原有基础人脸识别、眨眼活体检测、智能防撞报警功能的前提下，新增摇头、嘴部开合、关键点动态运动检测、光流法防翻拍等活体检测功能，提升人脸识别的安全性和准确性，确保系统识别的是真实人员，避免恶意破解或虚假验证，同时不影响原有防撞报警功能的正常运行，保障系统整体稳定性和实用性。

升级范围：系统人脸识别模块、活体检测模块，不涉及智能防撞报警模块的核心功能修改，仅同步适配人脸识别模块的升级，确保两大模块协同运行；同步完善工程目录、部署流程及相关文档，适配升级后功能。

<a name="heading_7"></a>**3.2 核心算法流程（升级后）**

1、活体检测维度：升级前仅支持眨眼检测；升级后新增摇头检测、嘴部开合检测、头部姿态（点头）检测、关键点动态运动检测、光流法防翻拍，形成多维度活体检测体系。

2、判定逻辑：任意3个维度满足阈值 + 连续8帧验证成功 → 判定为活体（兼容升级前眨眼检测逻辑，可单独开启或组合开启）。

3、防攻击能力：升级前仅能抵御静态照片攻击；升级后可有效抵御静态照片、电子屏翻拍、短视频攻击，安全性大幅提升。

<a name="heading_11"></a>**4.2 部署步骤（升级适配说明）**

<a name="heading_12"></a>**4.2.1 升级前部署（V1.0，仅眨眼活体检测）**

<a name="heading_13"></a>**方式1：手动部署**

1、解压face\_live\_detect\_system\_v1.0工程包到本地目录（路径不含中文/特殊字符）

2、下载 `shape\_predictor\_68\_face\_landmarks.dat` 模型文件，放入工程根目录

3、打开终端/命令提示符，进入工程目录

4、安装依赖：`pip install -r requirements.txt`

5、运行主程序：`python main.py`

6、按 `q` 退出，按 `r` 重置活体验证状态（仅验证眨眼动作）

<a name="heading_14"></a>**方式2：一键部署（推荐）**

Windows：双击运行 `deploy\_windows.bat`

Linux：终端执行 `bash deploy\_linux.sh`

脚本自动完成：创建目录→安装依赖→运行主程序→执行眨眼检测测试→生成测试报告

<a name="heading_15"></a>**4.2.2 升级后部署（V2.0，新增摇头等多维度活体检测）**

<a name="heading_16"></a>**方式1：手动升级部署**

1、备份V1.0工程包及数据（人脸库、测试记录等）

2、解压face\_live\_detect\_system\_v2.0工程包，替换原有V1.0目录（或新建目录部署）

3、确保模型文件 `shape\_predictor\_68\_face\_landmarks.dat` 已放入工程根目录（与V1.0通用）

4、进入工程目录，执行 `pip install -r requirements.txt`（更新适配新增算法的依赖）

5、将“截图文件夹”放入工程根目录，确保截图命名规范、可正常查看

6、运行主程序：`python main.py`

7、按 `q` 退出，按 `r` 重置活体验证状态（可验证眨眼、摇头等多维度动作）

<a name="heading_17"></a>**方式2：一键升级部署（推荐）**

1、Windows：双击运行V2.0工程包中的 `deploy\_windows.bat`，脚本自动完成升级、依赖更新、测试。

2、Linux：终端执行V2.0工程包中的 `bash deploy\_linux.sh`，脚本自动完成升级部署。

3、脚本额外完成：对比V1.0版本差异、迁移原有数据、生成升级报告、同步放置截图文件。

<a name="heading_19"></a>**5.1 启动流程（升级前后一致）**

1、确保摄像头已连接且未被其他程序占用；

<a name="heading_20"></a>**5.2 活体验证操作（升级前后对比）**

<a name="heading_21"></a>**升级前（V1.0）**

1、面对摄像头，保持面部在画面中央

2、完成眨眼动作，界面实时展示眨眼检测得分（0/1）

3、连续8帧眨眼检测通过 → 显示「✅ 活体验证通过」

4、若验证失败，系统自动重置，可重新验证

（嵌入截图：V1.0\_眨眼检测\_01.png，标注“升级前：仅眨眼动作提示界面”）

<a name="heading_22"></a>**升级后（V2.0）**

1、面对摄像头，保持面部在画面中央；

2、完成任意组合动作（眨眼+张嘴+轻微摇头/点头），界面实时展示各维度检测得分（0/1）；

3、连续8帧满足≥3个维度得分 → 显示「✅ 活体验证通过」；

4、可在系统设置中切换检测模式（仅眨眼检测/多维度检测），适配不同场景需求；

5、若验证失败，系统自动重置，可重新验证，界面提示具体失败维度；

（嵌入截图：V2.0\_多维度检测\_01.png，标注“升级后：多维度动作提示及得分展示界面”）。

<a name="heading_24"></a>**5.4 常见问题处理（升级后补充）**

|问题现象|原因|解决方法|
| :-: | :-: | :-: |
|报错 `Unable to open shape\_predictor\_68\_face\_landmarks.dat`|模型文件缺失/路径错误|下载模型文件放入工程根目录，核对文件名|
|摄像头无法打开|权限不足/被占用|关闭其他占用摄像头的程序，以管理员身份运行|
|活体检测准确率低|光线过暗/动作幅度过小/检测模式设置不当|调整环境光线，增大动作幅度，切换至多维度检测模式|
|运行卡顿|硬件性能不足/多维度检测占用资源|降低摄像头分辨率，切换至仅眨眼检测模式，提升硬件配置|
|摇头动作无法检测|动作幅度过小/头部超出采集框|保持头部在采集框内，增大左右摇头幅度（幅度适中即可）|

<a name="heading_28"></a>**6.3 测试总结**

本次测试覆盖升级前后系统的核心功能（活体检测、人脸识别、智能防撞报警），共测试13个测试项，全部通过测试。升级前（V1.0）系统仅支持眨眼活体检测，可满足基础身份核验需求，但防翻拍能力较弱；升级后（V2.0）系统新增摇头、嘴部开合等多维度活体检测功能及光流法防翻拍功能，核心性能指标均满足设计要求，活体检测准确率、防攻击能力大幅提升，运行稳定。

升级后系统兼容原有眨眼检测模式，单帧处理耗时和连续验证耗时仍在合理范围，与人脸识别模块、智能防撞报警模块。
